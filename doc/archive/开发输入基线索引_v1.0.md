## 3. 文档优先级规则

1. 需求范围冲突时，以《产品需求说明书 v0.3》为准。
2. 架构边界冲突时，以《系统架构设计 v0.3》为准。
3. 接口契约冲突时，以《接口文档 v0.2》为准。
4. 权限自然语言描述与权限矩阵冲突时，以《权限判定矩阵 v1.0》为准。
5. DEV 任务接口归属冲突时，以《任务-接口精确映射表 v1.0》为准。
6. 完成标准冲突时，以《验收清单终版 v1.2》为准。
7. 技术实现细节冲突时，以《详细技术方案 v0.4》为准，但不得突破 PRD 和架构约束。

## 5. 自动开发输入规则

1. 自动开发工具必须读取本索引列明的 P0 文档。
2. 不得使用旧版 PRD、架构、技术方案或验收清单作为开发依据。
3. 不得自行扩展 PRD 非目标范围。
4. 不得实现未在接口文档中定义的业务 API，除非先更新接口文档。
5. 不得绕过适配层直接绑定 AI、对象存储、文件扫描、报告引擎、检索、邮件、队列或认证厂商 SDK。
6. 所有接口实现必须遵守权限判定矩阵。
7. 所有核心 API 必须能通过任务-接口精确映射表回溯到 DEV 任务。
8. 所有开源治理和发布准入项必须追溯到 NFR-016 + ENG-OSG-001。
