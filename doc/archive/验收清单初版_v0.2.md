# 验收清单初版 v0.2

**产品名称：AI 原生本科专业建设与学生能力成长平台**  
**生成依据：《产品需求说明书 v0.2》、AC-DEC-001～AC-DEC-014 验收决策**  
**文档角色视角：QA 负责人 + 产品验收负责人**  
**文档定位：将功能需求与非功能需求转化为可判断通过 / 失败的验收项，供产品验收、测试设计、开发自测和阶段发布评审使用。**  
**版本说明：本版已回填 14 项验收待确认问题的确认结果，当前无阻塞性待确认验收项。**

## 1. 输入充分性检查

**结论：输入足够生成《验收清单 v0.2》。**

本清单覆盖《产品需求说明书 v0.2》中的 **FR-001～FR-032** 和 **NFR-001～NFR-015**。每条验收项均关联至少一个需求编号，并包含前置条件、操作步骤、预期结果、验收方式、优先级和确认状态。

本版已回填 **AC-DEC-001～AC-DEC-014**，原 14 条待确认验收项均已转为明确验收规则。

## 2. 验收优先级定义

| 优先级 | 含义 | 阶段验收建议 |
|---|---|---|
| P0 | 发布阻断项 / 安全红线 / MVP 核心闭环 | 对应阶段必须 100% 通过，否则不建议验收通过。 |
| P1 | 重要功能与重要质量项 | 对应阶段应通过；未通过需明确缺陷等级、修复计划和风险接受人。 |
| P2 | 后续阶段或增强场景验收项 | 到达对应 MVP 阶段时执行；前序阶段可作为预验收参考。 |

---

## 3. 验收方式定义

| 验收方式 | 说明 |
|---|---|
| 功能测试 | 通过页面或接口验证功能是否按需求完成。 |
| 权限测试 | 使用不同角色账号验证可见范围、操作范围和越权拦截。 |
| 流程测试 | 验证状态流、审核流、发布流、授权流等是否正确流转。 |
| 异常流程测试 | 验证不支持格式、越权、数据缺失、重复数据、撤销授权等异常场景。 |
| 数据一致性核查 | 对比页面、导出文件、数据库记录、版本记录和审计日志。 |
| 安全 / 合规测试 | 验证学生隐私、AI 安全红线、版权许可、对外授权和审计要求。 |
| 性能测试 | 验证页面响应、异步任务和报告生成等非功能指标。 |
| 兼容性测试 | 验证文件格式、PC Web、移动 H5、导出文件打开兼容性。 |

## 5. 建议测试数据基线

以下测试数据用于验收准备，不构成新增产品范围。

### 5.1 功能验收最小数据

| 数据类型 | 建议准备 |
|---|---|
| 学校 | 1 个测试学校，按单学校独立空间 / 单学校独立租户准备。 |
| 学院 | 至少 2 个学院。 |
| 专业 | 至少 3 个专业，其中 1 个专业完整跑通专业建设闭环。 |
| 用户角色 | 系统管理员、学校管理者、学院管理者、专业负责人、教务、教师、导师、学生、企业导师、知识库管理员。 |
| 课程 | 至少 10 门课程，覆盖不同模块、学期、学分、理论 / 实践学时。 |
| OBE 数据 | 至少 2 个培养目标、4 条毕业要求、8 个能力指标点。 |
| 支撑矩阵 | 至少包含高 / 中 / 低 / 无四级支撑关系。 |
| 知识库资料 | docx、pdf、xlsx、md、txt 和 1 条手动 URL。 |
| 学生数据 | 至少 10 名学生、课程修读记录、学习证据、导师关系。 |
| 企业数据 | 至少 1 个用人单位、1 个企业导师、1 个岗位能力模型。 |

### 5.2 性能验收数据量基线

| 阶段 | 数据量基线 |
|---|---|
| MVP-1 | 1 个学校；不少于 2 个学院；不少于 3 个专业；不少于 1 个完整专业建设闭环；不少于 100 门课程；不少于 300 条能力指标点；不少于 1,000 条课程支撑矩阵关系；不少于 10 个培养方案 / 课程体系版本记录。 |
| MVP-2 | 不少于 300 份知识库资料；不少于 3,000 个知识片段；不少于 100 条引用记录。 |
| MVP-3 | 不少于 1,000 名学生；不少于 10,000 条学习证据或课程修读记录。 |
| MVP-4 | 不少于 100 份能力地图；不少于 50 个用人单位 / 岗位能力模型记录。 |

### 5.3 基础并发基线

| 并发类型 | 验收基线 |
|---|---|
| 在线用户 | 20 个并发在线用户。 |
| AI 任务 | 5 个并发 AI 任务。 |
| 文件解析 / 报告生成 | 5 个并发文件解析或报告生成任务。 |

## 9. 验收红线

以下任一项失败，建议判定对应阶段验收不通过：

- AI 直接发布正式培养方案、课程体系、课程内容或学生能力结论。
- 未授权角色可访问学生画像、成长预警、申诉记录或未授权能力地图。
- 未审核学习证据被计入正式能力等级。
- 学生能力升级在无证据情况下生效。
- 未审核、未授权或已禁用知识库资料被用于正式课程内容发布。
- 课程体系调整覆盖历史已发布版本。
- 关键发布、审核、导出、授权、归档、恢复行为没有审计日志。
- 已发布关键版本、AI 运行记录、引用记录、审核记录或审计日志被物理删除。
